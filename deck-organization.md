# Deck list organization (EvalDragons)

Use these **H2 section headers** in `decks/*.md` so agents can **count core-ratio buckets** without reinventing categories.

## Required sections (map to ratios)

1. **Commander** — commander card(s); not part of the 99 counts.
2. **Lands**
3. **Ramp** — mana acceleration (rocks, dorks, land ramp).
4. **Card draw** — burst and steady sources.
5. **Targeted removal** — spell/permanent-based answers to specific threats.
6. **Board wipes** — mass removal / resets.
7. **Synergy / strategy** — theme pieces, payoffs, win lines, plan tutors.

## Optional

8. **Utility / MDFC** — catch-all for cards that are mostly utility, modal DFCs played as lands, or genuinely dual-role pieces (note primary bucket in parentheses if helpful).

## Deck note template

Each deck file should include:

- `Source: <Moxfield or Archidekt URL>` (or pasted export path).
- Optional YAML frontmatter: `target_bracket: 2` (1–5, Wizards Commander Brackets).

Mis-grouped cards (e.g. draw under Lands) should be called out in reviews.
